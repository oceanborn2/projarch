Document properties

  • Primary maintainer : This may change over time but the full list must be kept.
  • Reviewers : It speaks itself. Again, there may be many.
  • Acceptance level : Is this document accepted by all involved parties, it is still consistent with the current issues...
  • Publication status :

    • Planned : The need for this document has been clearly expressed
    • Draft : This document is being written but is clearly at an early stage.
    • Released/Published : Users other than the author itself can consult the documents. There will be refinements in the ways a document can be published:

      • restricted publication : Only a subset of the users will be allowed to read the document. These users may be defined in terms of groups.
      • internally published : All the users can read the document.
      • publically published : Absolutely no restrictions are imposed on the publication of the document
    • Revised : Documents are often revised. Several kinds of revisions may be implemented :

      • full revisions : Basically, a major (if not near complete) rewrite of the document. Previous versions of the document will be kept but will not remain directly visible to the user.
      • partial revisions : These revisions will include patches to the original document. Control versionning and managed document formats will be important here.
  • Distribution lists : To automate distribution of the documents
  • Security : Security information (see above).
  • Revision control information : This will be different from CVS style version management for the following reasons :

    • We want to have a fine-grained control over the kind of information kept under control. Obviously, file level is too limited.
    • We also want to be able to move some piece of information (or document) around without having to go into CVS like manipulations (removing from a directory/folder, adding in another, then committing). This should be done transparently, at least from the user point of view (It may be done like CVS internally) : One must be able to drag and drop items easily.
  • Node structure : A tree depicting the document structure. XML is one of the cool language for doing just that, but there are others. Not all data formats will allow this (proprietary formats, monolithic formats).
  • IsFolder : Is the document a folder (containing other documents and folder => May be handled via node structure).
