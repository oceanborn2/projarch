This is very basic right now, but it will get you up and running.

For all developer (read/write) access, you will need to use SSH. SSH (1.x) client must be available on your local machine. The local environment variable CVS_RSH must also be set to the path to ssh. This is done on most unix (bash) systems by typing:

    export CVS_RSH=ssh

Anonymous CVS access uses CVS pserver and does not require SSH.

If you get 'permission denied' errors with no prompt for a password, you do not have this environment variable set properly or SSH is not available to your system. Fix this before suspecting a password problem.

What to do FIRST
SSH into your user account on the CVS server. This will create your home directory so that you can begin to use CVS. CVS access will NOT work until you have done this.

How to import source code into your repository

  • On your local machine, change to the directory whose files (and subdirectories) you want to import. Everything now in the current directory and all subdirectories will be imported into the tree.
  • Type the following, where loginname is your SourceForge login name, yourproject is the unix group name of your project, and directoryname is the name of the new CVS root level directory.
    cvs -d:ext:loginname@cvs.yourproject.sourceforge.net:/cvsroot/yourproject import directoryname vendor start

How to check out source via SSH

  • Type the following, making necessary obvious substitutions for your username and project.
    cvs -d:ext:loginname@cvs.yourproject.sourceforge.net:/cvsroot/yourproject co directoryname
  • After initial checkout, you can change into this directory and execute cvs commands without the -d tag. For example:
    cvs update
    cvs commit -m "comments for this commit"
    cvs add myfile.c

How to check out source anonymously through pserver

  • Type the following, making necessary obvious substitutions for your username and project.
    cvs -d:pserver:anonymous@cvs.yourproject.sourceforge.net:/cvsroot/yourproject login
  • After anonymously logging in:
    cvs -z8 -d:pserver:anonymous@cvs.yourproject.sourceforge.net:/cvsroot/yourproject co directoryname
  • After initial checkout, you can change into this directory and execute cvs commands without the -d option. For example:
    cvs update

How to use cvs when logged on shell.sourceforge.net

  • Type the following, making necessary obvious substitutions for username and project.
    cvs -dloginname@cvs1:/cvsroot/yourproject co directoryname
See CVS/SSH FAQ for more information.

How to import an existing CVS tree

This must be done manually by the SourceForge staff. Proceed as follow:

  • Prepare a tar/gzip of your entire CVS tree.
  • Make sure you include the CVSROOT directory if it contains information you want to keep. If you're not sure, include it.
  • Submit a support request asking them to include the tar/gzip in the CVS repository of your project. Make sure you provide the necessary information in your request:
    • The exact name of the project or the URL of its home page. Something like http://sourceforge.net/project/?group_id=1695.
    • The ftp or http location of the tar/gzip file, or its location on shell1. Double check with your navigator that the file is available.
    • Your SourceForge login name.
    • Your project's unixname
    • Your mail address.
  • You will be mailed by the SourceForge staff when the CVS tree is ready for use.

How to get your CVS tree back

Daily tarballs of entire project CVS Repositories are available for backup or mirroring purposes. These are available at: http://cvs.sourceforge.net/cvstarballs/projectname-cvsroot.tar.gz. Where projectname is the Unix name of your project.
