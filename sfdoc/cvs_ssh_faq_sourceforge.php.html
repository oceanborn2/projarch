This FAQ is intended to help using CVS with SSH without reading the complete CVS and SSH documentation.

  • How does CVS/SSH work?
  • When I try to use CVS it tells me that "/home/users/[account name]" does not exist.
  • Why does cvs hangs forever?
  • Am I forced to enter my password for every cvs command?
  • I updated my CVS/SSH shared key but I'm still asked for a passphrase?
  • CVS does not work on shell.sourceforge.net?

How does CVS/SSH work?

SSH is the authentication mechanism used to allow authorized users to have read/write access to the CVS repository. The protocol used is ssh protocol version 1. You must not try to use ssh protocol version 2. If you have both ssh1 and ssh2 installed on your machine, the ssh1 commands are all suffixed with a 1. For instance use ssh1 instead of ssh or ssh-keygen1 instead of ssh-keygen.

CVS will use SSH for authentification if you set the environment variable CVS_RSH to ssh. CVS_RSH=ssh or CVS_RSH=ssh1.

When I try to use CVS it tells me that "/home/users/[account name]" does not exist.

You need to ssh into your account on the CVS server once before attempting to use CVS for the first time. Sshing in will cause your home directory to be created.

Why does cvs hangs forever?

If you're behind a firewall you may need to instruct ssh not to use a privileged port. Insert the
UsePrivilegedPort no
line in your ~/.ssh/config file. This is equivalent to specifying the -P option on the command line. You need to do that since you can't simply set CVS_RSH='ssh -P'.

Am I forced to enter my password for every cvs command?

No, you can upload a shared key that will allow SSH sessions to be established without the need to enter you password every time. Go to your Account Maintainance page, click on the Edit Keys link and follow the instructions.

I updated my CVS/SSH shared key but I'm still asked for a passphrase?

A passphrase is not a password. When you used ssh-keygen (or ssh-keygen1) to produce your shared key, you were asked for a passphrase. There are two possibilities to avoid cvs passphrase prompts: you can use ssh-agent tool or you can generate key pair without passphrase.

To use ssh-agent edit your .xsession (or .xstart) script and change line where window manager is started into:

ssh-agent $WM

where $WM is your window manager. Logout-login to activate the ssh-agent. Use ssh-add to add your key to the ssh-agent (you must enter the passphrase). As long as the ssh-agent runs you will not be prompted for the passphrase again. For more details see the man pages of ssh-agent and ssh-add.

To generate unprotected key pair just type return at the ssh-keygen passphrase prompt. The drawback of the unprotected key pair is that everybody who has your private key will have access to the cvs repository.

CVS does not work on shell.sourceforge.net?

When you're logged on shell.sourceforge.net using ssh1, you should use the cvs1 hostname instead of the full host name. For instance:

  • Not good: cvs -d:ext:precision@cvs.phpsysinfo.sourceforge.net:/cvsroot/phpsysinfo -z3 co phpsysinfo
  • Good: cvs -d:ext:precision@cvs1:/cvsroot/phpsysinfo -z3 co phpsysinfo
This has to do with the fact that both the shell servers and the cvs servers are firewalled behind the same firewall. Since you're inside the firewall when you're logged on shell.sourceforge.net, the rules do not apply in the same way.
