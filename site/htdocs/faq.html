Frequently asked questions page

Question: Is the Project Architect an Integrated development environment ?
Answer:nope. Basically, the Project Architect is a project management system. It is intended to help automate all kinds of activites related to this topic : Requirements capture, documentation management and publication ... See to foundation documents to get more information. In the future (far from now), some code generation could be integrated by the mean of plugins, but this is neither a priority nor a primary feature for this project.

Question: The cathedral and the bazaar project style. Is it compatible with a project management system ?
Answer:yes. Even tough typical free software and open-source projects are very active and subject to codes changes from lots of developpers with varying needs and interests, using a project management system can help track down features and changes. [develop more]

Question: What kind of tools are available to the users ?
Answer:Not all the features are decided as of today but features should comprise document management (handling, storing, indexing and searching, requirements capture, ressources scheduling, a workflow system ... There is still a lot of brainstorming to be done at this stage
